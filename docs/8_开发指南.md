# DEV_GUIDE: 开发指南

## 1. 常用命令 (Common Commands)

### 开发 (Development)
- **启动开发服务器**:
  ```bash
  npm run dev
  ```
- **构建生产版本**:
  ```bash
  npm run build
  ```
- **运行 Lint 检查**:
  ```bash
  npm run lint
  ```

## 2. 编码规范 (Coding Standards)

### 2.1 命名规范
- **文件/目录**: 使用 `kebab-case` (如 `task-card.vue`, `components/ui/`)。
- **组件**: 使用 `PascalCase` (如 `TaskCard`, `UserProfile`)。
- **函数/变量**: 使用 `camelCase` (如 `fetchTasks`, `isLoading`)。
- **类型/接口**: 使用 `PascalCase` (如 `Task`, `UserSettings`)。

### 2.2 代码风格与质量 (Code Quality)
- **代码可读性**:
  - **严禁**为了压缩文件体积将代码写在一行。
  - 代码必须保持良好的格式化和缩进，变量命名必须语义化。
- **注释**: 所有注释必须使用 **中文**。
- **类型安全**: 严禁使用 `any`，必须定义明确的 Interface 或 Type。

### 2.3 Vue 最佳实践
- **DOM 操作**:
  - **禁止**使用原生 JavaScript 直接操作 DOM (如 `document.getElementById`, `innerHTML` 等)。
  - 必须通过 Vue 3 的响应式机制 (`ref`, `reactive`, `v-model`) 来驱动视图更新。
- **组件结构 (Script Setup)**:
  ```vue
  <script setup lang="ts">
  // 1. 导入
  import { ref, onMounted } from 'vue';
  import { useStore } from '@/stores';
  import TaskCard from '@/components/TaskCard.vue';

  // 2. 类型定义
  interface Props {
    title: string;
  }

  // 3. Props & Emits
  const props = defineProps<Props>();
  const emit = defineEmits(['update']);

  // 4. 状态 (State)
  const count = ref(0);

  // 5. 方法 (Methods)
  const increment = () => {
    count.value++;
  };
  </script>

  <template>
    <div class="container">
      <TaskCard :title="props.title" />
      <button @click="increment">{{ count }}</button>
    </div>
  </template>

  <style scoped>
  /* 样式 */
  .container {
    padding: 20px;
  }
  </style>
  ```

## 3. 安全与隐私 (Security)
- **零信任前端**: 永远不要在前端代码中硬编码 API Key 或敏感 Secret。
- **输入清洗**: 假设所有用户输入都是恶意的，必须进行校验。
