# QA_CHECKLIST (发布前验证清单)

在每次发布或合入主分支前，请务必按照以下清单进行自我核查。

## 1. 移动端适配 (Mobile First)
- [ ] **真机测试**: 必须在手机浏览器上实际打开页面，而不是仅依赖开发者工具的模拟器。
- [ ] **布局堆叠**: 确认多列布局在小屏幕上是否正确转为单列堆叠。
- [ ] **触控区域**: 确认按钮和链接的点击区域足够大 (至少 44x44px)。
- [ ] **导航菜单**: 确认汉堡菜单能否正常展开和关闭，且不遮挡核心内容。

## 2. 边界状态 (Edge Cases)
- [ ] **空状态 (Empty State)**: 
  - 任务列表为空时，是否有友好的插画或提示？
  - 搜索结果为空时，是否有“未找到”的反馈？
- [ ] **加载状态 (Loading State)**:
  - 数据请求期间是否有 Skeleton (骨架屏) 或 Spinner (加载圈)？
  - 按钮点击后是否禁用并显示 loading 状态，防止重复提交？
- [ ] **错误状态 (Error State)**:
  - 断网或 API 失败时，是否有 Toast 或 Alert 提示？
  - 表单校验失败时，是否有红色的错误文字提示？

## 3. 交互健壮性 (Robustness)
- [ ] **狂点测试**: 快速连续点击提交按钮，确认不会触发多次 API 调用。
- [ ] **长文本溢出**: 输入超长标题或描述，确认 UI 不会崩坏 (应截断或换行)。
- [ ] **特殊字符**: 测试 Emoji 表情和特殊符号的显示与存储。

## 4. 安全与隐私 (Security)
- [ ] **密钥检查**: 打开浏览器开发者工具 (F12) -> Network，确认请求头或响应中没有暴露 API Key 或敏感信息。
- [ ] **输入验证**: 确认所有表单输入都进行了基本的格式校验。

## 5. 性能 (Performance)
- [ ] **图片优化**: 确认大图是否使用了 Next.js `<Image>` 组件或进行了压缩。
- [ ] **CLS (累积布局偏移)**: 确认图片加载前后页面内容没有剧烈跳动。
