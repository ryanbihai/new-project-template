# Vibe Coding 项目启动指南

欢迎使用这套“文档驱动开发”的模板。这不仅仅是一些文件，而是一套让 AI (Trae) 为你高效工作的**协作系统**。

## 1. 快速开始 (3步走)

### 第一步：下载与打开
1. 将本项目解压到你电脑上的任意文件夹。
2. 打开 **Trae IDE**。
3. 选择 `File -> Open Folder`，打开这个文件夹。

### 第二步：告诉 AI 你要干什么
1. 打开 Trae 的 **Builder** 模式（右侧对话框）。
2. 在输入框中输入：
   > “我想做一个[你的项目名]，请启动 doc-architect 帮我梳理需求。”
3. AI 会通过 Skill 引导你回答问题，并自动生成/更新 `docs/` 下的业务文档。

### 第三步：开始 Vibe Coding
1. 当文档准备好后，对 AI 说：
   > “请阅读 `1_产品需求.md`，按照 `6_实施计划.md` 的第一步开始构建。”
2. AI 写完代码后，记得运行一下看看效果（在终端输入 `npm run dev`）。

---

## 2. 文件夹里有什么？

这个模板包含了你和 AI 协作所需的一切：

- **`.traerules`**: 這是 Trae 的“家规”，告诉它必须写中文注释、必须先读文档。
- **`.trae/skills/`**: 这里藏着一个“文档架构师 (doc-architect)”，它是你的私人产品经理助理。
- **`docs/`**:
  - `3_技术栈.md` & `5_数据结构.md`: **不要动**，这是 IT 部门配置好的“地基”。
  - `1_产品需求.md` 等：这是你的画布，尽情修改。
- **`进度跟踪.md`**: 每次 AI 干完活，记得让它更新这个文件，这样下次打开它就不会“失忆”。
- **`避坑指南.md`**: 如果 AI 犯错了，纠正它之后让它记在这里，下次它就不会再犯。

## 3. 注意事项
- **不要删除隐藏文件**：`.traerules` 和 `.trae` 文件夹是系统运行的核心，请勿删除。
- **常更新进度**：养成习惯，每次对话结束前说一句“**总结进度到 进度跟踪.md**”。
