# 产品需求 (PRD) - 智能理赔助手 Demo

## 1. 项目愿景
打造一个“小白友好”的理赔材料整理工具。针对普通用户或初级保险代理人，解决“因材料提交错误/缺失导致理赔失败”的痛点。
**核心价值**：用户只管上传，AI 负责筛选、分类和纠错。

## 2. 目标用户
- **普通投保人**：不懂保险条款，不知道该交什么发票或病历。
- **初级代理人**：需要协助客户整理材料，但经验不足。

## 3. 核心功能 (MVP Demo)
- **智能材料上传**：支持批量上传图片（发票、病历、检查报告、身份证等）。
- **AI 智能分类与筛选**：
  - **自动分类**：识别图片内容，自动归类为“身份证明”、“医疗发票”、“病历资料”等。
  - **有效性判断**：标记“有效材料”（如正式发票）和“无效材料”（如无关的生活照、模糊不清的图）。
  - **缺失提醒**：基于理赔类型（如医疗险），提示用户“缺少出院小结”或“缺少发票联”。
- **理赔清单生成**：生成一份清晰的“理赔材料提交清单”，用户可预览并确认。
- **数据保存**：所有数据暂时保存为本地 JSON 文件（模拟后端存储）。

## 4. 成功标准
- 用户上传 5 张混乱的图片（包含 1 张无关图），系统能准确剔除无关图。
- 系统能正确提示“缺少身份证正面”。
- 整个流程在 3 分钟内完成。

## 5. 非功能性需求
- **隐私保护**：Demo 阶段图片仅在浏览器本地处理（模拟），不上传服务器。
- **响应速度**：图片识别反馈时间 < 2秒（模拟 AI 延迟）。
- **移动端适配**：主要在手机浏览器上使用。
